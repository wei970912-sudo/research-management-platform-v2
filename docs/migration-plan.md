# 迁移规划

## 迁移原则

- 旧版目录不动
- 新版独立开发
- 先迁数据模型，再迁页面功能
- 先迁核心模块，再迁附属能力

## 建议优先级

1. 项目中心
2. 进度与里程碑
3. 会议记录
4. 合同与经费
5. 文档归档
6. 日历提醒
7. 总览驾驶舱

## 数据迁移思路

旧版主要数据来自浏览器 `localStorage`，建议后续新增迁移工具：

- 导出浏览器本地 JSON
- 校验字段完整性
- 映射到新版数据库表
- 生成导入日志

## 风险点

- 旧版字段命名不统一
- 模块间通过存储键名隐式耦合
- 部分中文内容存在编码异常
- 统计逻辑散落在前端，迁移时需要下沉到后端服务层
